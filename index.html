01

NoteD&DCharacter

Aplicación web interactiva para importar, consultar y usar fichas de Dungeons & Dragons durante una partida real. Permite importar personajes desde Nivel20/PDF, persistir datos y gestionar inventario, cartera, munición, salud, escudo, estados, conjuros y rasgos por usuario.

Ver explicación del proyecto

El objetivo del proyecto es que una ficha de D&D sea útil durante una partida, no solo una copia visual del PDF. La aplicación separa información estable del personaje, herramientas rápidas de combate y datos editables durante la sesión.

Trabaja con importación desde Nivel20 y PDF, parser propio para extraer datos, persistencia en Supabase y estado editable por usuario para no pisar vida, vida temporal, espacios de conjuro o munición de otras cuentas.

Incluye gestión de inventario, cartera, munición, salud, escudo, estados activos, conjuros, rasgos y descripciones desplegables. Es un proyecto donde se mezclan frontend, lógica, datos, APIs internas y diseño de una interfaz pensada para uso real.

03

Papelera inteligente

TFC de prototipado físico. Mi parte se centró en convertir la detección del residuo en movimiento real: diseño 3D de piezas, bloque interno, embudo-engranaje, motores, servos, sensor y control mediante Arduino.

Ver explicación y vídeos

El proyecto combinaba una parte de detección mediante cámara e IA con una parte física capaz de mover el residuo hacia el compartimento correspondiente. Mi responsabilidad principal fue la parte mecánica y de control: diseñar piezas funcionales, preparar el mecanismo y coordinar motores, servos y sensor con Arduino.

Diseñé el bloque interno que ocultaba servos, motores y cables, además del embudo, el engranaje y los dientes del embudo para convertirlo en una pieza mecánica funcional. La idea era que la detección digital terminase produciendo una acción física real.

04

Galería de cuadros

Web informativa conectada a base de datos para visualizar cuadros y gestionar contenido desde administrador, con creación y edición de datos.

Ver explicación del proyecto

La galería está pensada para consultar información de cuadros de forma ordenada, separando la parte pública de visualización y la parte de administración de contenido.

El administrador puede añadir nuevos registros o editar datos existentes, lo que convierte el proyecto en una práctica completa de gestión de contenido: estructura web, base de datos, formularios y operaciones CRUD básicas.

Este proyecto demuestra una parte más orientada a lenguajes de marcas, interfaces sencillas y organización de información persistente.
